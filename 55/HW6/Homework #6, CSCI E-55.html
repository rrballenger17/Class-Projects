Homework #6 is different from previous homeworks in that you are not required to write any new Java code. Instead, you are instructed to install Hadoop on your system, pull down some data files and a Java file that encapsulates a Map/Reduce solution, and run it using the Hadoop API's, targeting the data you pulled down.

Submission requirements:

  • A journal describing your steps along the way, including outputs, successful and otherwise, from the start, getting the Hadoop tarball, to running WordCount locally and on the HDFS.
  • Two part-r-00000 files generated by WordCount, running locally, on two data sets:
    • the very small data set contained in fleas.zip, and,
    • the file contained in jabberwocky.zip

Extra credit: Copy the data into the HDFS and run WordCount targeting your running Hadoop (pseudo-distributed) cluster. Submit jabber-tail.out as below.

Steps on the way to running on your cluster:

  • $ hadoop fs -mkdir -p /user/$USER/input
  • $ hadoop fs -copyFromLocal jabberwocky.txt input
  • $ hadoop fs -ls input # Verify the file is in its place
  • $ jar cf wordcount.jar WordCount*.class
  • $ hadoop jar wordcount.jar WordCount input output
  • $ hadoop fs -tail output/part-r-00000 | tail > jabber-tail.out
