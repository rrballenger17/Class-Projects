    Booster - Validation against RNG, RNC, XSD

    Re-submit Assignment
    • Due Nov 24, 2015 by 12:15pm
    • Points 20
    • Submitting a text entry box

    With this booster, you'll validate three XML files using different schema definitions (RNG, RNC, XSD).  Let us know which files validate and which ones do not validate.

    You are provided the booster-validation.zip file. Download this and unzip it:

    booster-validation
|-- people-model-illustrated-rng.png
|-- people-model-illustrated-xsd.png
|-- people.rnc
|-- people.xsd
|-- people1.xml
|-- people2.xml
`-- people3.xml

    XML definition for People. Note that the RNG, RNC, and XSD express the same definition for XML. Screenshots of the documentation produced by Oxygen for the RNG/RNC and XSD are shown:
    RNG/RNC

    XSD

    Validation. Note that two of them will pass validation and one will fail. Before doing the technical work, I encourage you inspect the XML documents themselves and the schema files and see if you can determine which ones are valid and which one will fail validation.

    Use Oxygen or command line tools to validate the people1.xml, people2.xml, people3.xml against these schemas.

    • Oxygen (recommended):  Validation options are accessed through the  "red check mark" icon within the editing window.  Under the "Tools" menu, you can access the "Generate/Convert Schema..." option that will allow you to produce a schema from XML, or convert from one schema to another.  


    • Jing.  Jing is a Java-based command-line tool that will validate using RELAX NG, either RNG or RNC.  See http://www.thaiopensource.com/relaxng/jing.html
    • Saxon - Running Validation (W3C XML Schema) from the Command Line
      http://www.saxonica.com/html/documentation/schema-processing/commandline.html

    Submission.  Tell us which files validated and which ones didn't -- e.g. "people99.xml valid; people100.xml valid; people101.xml failed; etc."
