    XSL-FO and SVG

    • Due Nov 13, 2015 by 12:15pm
    • Points 200

    Assignment 3 focuses on creating print (PDF via XSL-FO) and graphics (SVG) content using XQuery, XSLT, and eXist.  The data is from course data from the FAS at Harvard which you have already worked with in Assignment 2.  An example of a single course file is at: SCIPHUNV 27 - The Science of Cooking

    This assignment is distributed via a GitHub repository, and you will need to submit the assignment via GitHub as well. The GitHub repository that has been created for you is in the form of jharvard-assignment3, where jharvard is your GitHub username.

    The repository contains essential files for your eXist application, including course data. You will need to clone your repository, build the xar file, deploy to eXist. Once you deploy, check it out in your browser — you should see a test page that we put in so you can confirm things are working!

    You'll then edit the project files, periodically synchronizing (exporting) the files from eXist back to your local repository.

    For this assignment, we have given you a completely functional controller.xql file as well as a working xquery/pdf.xq file (this does the transform and rendering).  You do not need to edit these at all, but feel free to inspect them to see what they are doing!

    Courses - PDF via XSL-FO

    Here you will author an XSLT (xsl/course-listing-fo.xsl) that will produce XSL-FO from course data.  Example PDF files are at (these aren't meant to be reproduced exactly, but to give an idea of what to aim for):

    • astronomy.pdf
    • gened.pdf

    You can also take inspiration from how course details are displayed from the official "my.harvard" site:

    Requirements and Suggestions

    • Your PDF should have a table of contents.
    • The table of contents entries should be linked internally to the course content.
    • Courses should be sorted based on course group (see the "Gened" PDF), then by numeric part of the short title, and lastly by section (@section).
    • About @section label:  Section labels are used to distinguish different classes of the same course. For an example, see "ASTRON 300" — a single course with many classes, each having different section labels within each term.  By default, section labels are "001" — these are typically not displayed unless there are other classes with the same course.  Compare SCIPHUNV 27 with ASTRON 300 for examples — the former has a section of "001" but is not displayed because there is only one class of the SCIPHUNV 27 course, whereas the latter has has several classes of the same course, so the section labels are displayed.
    • Instructor names should be sorted by @role and then by @seniority_sort (both are data-type="number")
    • Your detailed course listing should include:
      • Title
      • Short title, section if appropriate
      • Instructor names
      • Term
      • Meeting days and times
      • Description
      • Notes
    • PDF should contain a header or footer that includes the page number and total number of pages (e.g. "Page X of Y")
    • Use xsl:attribute-set to define sets of styles you will use
    • Page break after you are done with the table of contents

    SVG Chart of Top 25 Courses by Enrollment

    The main purpose of this exercise is to write XQuery that produces an SVG graph of the "Top 25 Largest Enrolled Courses". You'll need to edit xquery/top25courses.xq and use the data in data/enrollment.xml.

    The PDF you will produce should look something like:

    Requirements

    • Top 25 courses by greatest "total enrollment" should be displayed.
    • XQuery should generate a valid SVG file.  You can validate the SVG files at
      https://validator.w3.org/#validate_by_input
    • Label each part of your chart with the course "short_title"
    • Include a label that contains the value of the total enrollment

    Hints

    • data( ) function.  To get the attribute value (as opposed to the attribute node itself), you'll need to use the data() function.
      See:  https://gist.github.com/dpheitmeyer/7a3d12aa94fb71a2e6b3 
    • Sorting. Create a $sorted-courses variable where you sort the enrollments by @total_enrollment (descending). Use this sorted course list to get the top 25
      See:  https://gist.github.com/dpheitmeyer/dbab9cff7662b0cf3bfc 
    • subsequence( ) function.  To grab the first 25 of your sorted list, you'll want to look at the subsequence() function.
      See:  https://gist.github.com/dpheitmeyer/dbab9cff7662b0cf3bfc 
    • Getting the "position".  Knowing the position will be helpful so you know how to position rectangle and labels for that course. For this, you'll want to use the at expression within the for of your FLWOR statement — e.g.
      for $course at $position in $sorted-courses ...
      so that you will have access to both $course and $position (starts with 1). Note that the position() function won't work in an XQuery iteration like it does within an XSLT iteration.
      See: https://gist.github.com/dpheitmeyer/d3115952822b5277fd4d and 
      https://gist.github.com/dpheitmeyer/932ff269aa0bae7f5e4a
