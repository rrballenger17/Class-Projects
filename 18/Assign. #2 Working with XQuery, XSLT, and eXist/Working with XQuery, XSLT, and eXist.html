    Working with XQuery, XSLT, and eXist

    • Due Oct 20, 2015 by 12:15pm
    • Points 200

    Assignment 2 focuses on creating a very basic course catalog site using XQuery, XSLT, and eXist.  The data is from the student information system for the Faculty of Arts & Sciences and shows the courses that are offered through the FAS at Harvard.  The data is delivered as a collection of files.  An example of a single course file is at: SCIPHUNV 27 - The Science of Cooking

    The Course Catalog site will present the user a list of departments. The department names will be linked to a department course page that displays a table of the courses offered for each course group within the department. 

    Structure and Design of Output. In this assignment, the output is described, but the exact structure and design is left as part of the assignment exercise. Some screenshots are provided for visual clarity of the descriptions given. Other than meeting the requirements given, you are not constrained by needing to reproduce the screenshots exactly.

    Page of departments.
    Suggested URL: /departments

    Page of course listings - example for Near Eastern Languages and Civilizations
    Suggested URL: /courses?department=NELC

    This assignment is distributed via a GitHub repository, and you will need to submit the assignment via GitHub as well. The GitHub repository that has been created for you is in the form of jharvard-assignment2, where jharvard is your GitHub username.

    The repository contains essential files for your eXist application, including course data. You will need to clone your repository, build the xar file, deploy to eXist. Once you deploy, check it out in your browser -- you should see a test page that we put in so you can confirm things are working!

    You'll then edit the project files, periodically synchronizing (exporting) the files from eXist back to your local repository.

    For each of the departments and course listings page types, you'll need to have an entry in the controller.xql file, and corresponding XQuery and XSLT files for the XML pipeline.

    Requirements and Suggestions

    • Department listing page
      • Use XQuery to generate departments data, XSLT to transform to HTML
      • A suggestion for the data format you may wish to generate from your XQuery is shown at:  https://gist.github.com/dpheitmeyer/f5abf443d3b53a2ec336
      • Departments should be sorted alphabetically by name
      • Note that some department codes have "unsafe" characters for URIs. For example, Earth and Planetary Sciences has a code of E&PS. In a URI, the & must be URI-encoded, e.g. /courses?departments=E%26PS.
        You'll want to use the encode-for-uri() XPath function to encode values for URIs.
      • Pay attention to efficiency when constructing your XQuery to get the list of departments. If you are finding your page takes > 10 seconds to load, you'll want to make sure your XQuery is not repeatedly making expensive queries.
    • Courses listing page
      • Use XQuery to generate departments data, XSLT to transform to HTML
      • Link back to the department listing page
      • Sort courses by Course Group and then by the numeric portion of the short title.  For an example of how to accomplish this, see:  http://cscie18.dce.harvard.edu/lecture_notes/2015/20150922/slide16.html
      • Display the short title, title, and term in the course listing.  You may choose to display other fields as well, as you wish.
    • General
      • Use xsl:import to put common items between your XSLT in a single file (I usually call this common.xsl, and at the very least, this contains an <xsl:template match="/"> that takes care of a consistent HTML page structure.
      • Use some form of CSS for consistent styling between pages. It doesn't need to be complex, but it needs to be there. Feel free to use a CSS framework such as Bootstrap, Unsemantic, Skeleton, etc -- but these are certainly not required. If you write your own CSS, we're looking for CSS with a half-dozen to a dozen rules specified to make your page look better than than it would without CSS
